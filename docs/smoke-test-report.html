Flow Diagrams

The key change is that Miden-origin deposits use public programmable notes instead of per-quote Miden accounts. The bridge watches for a valid note, consumes it on Miden testnet, then releases or refunds on the EVM side.

Inbound is the reverse: after the EVM deposit confirms, the Bridge API uses the configured Miden solver account to create a public P2ID note targeted to the recipient. The bridge marks the quote successful once that mint transaction commits; the recipient completes wallet-side settlement by consuming the public note.

Solution overview

flowchart LR
    Old["Old plan: per-quote Miden account"] --> Problem["Problem: account is not visible until it has sent a tx"]
    Problem --> Pivot["Design pivot"]
    Pivot --> Note["Public BridgeOutV1 note"]
    Note --> Storage["NoteStorage commits bridge account and quote hash"]
    Storage --> Consume["Bridge account consumes note on Miden testnet"]
    Consume --> Settle["Release, refund, or mark final status"]

Inbound: EVM deposit to Miden payout

sequenceDiagram
    participant User
    participant Anvil
    participant BridgeAPI as Bridge API
    participant Miden
    participant Recipient

    User->>BridgeAPI: Request quote eth-anvil:eth to miden-testnet:eth
    BridgeAPI-->>User: Return EVM deposit address
    User->>Anvil: Send deposit tx
    BridgeAPI->>Anvil: Detect and confirm deposit
    BridgeAPI->>BridgeAPI: Move quote to PROCESSING
    BridgeAPI->>Miden: Submit solver-signed public P2ID mint to recipient
    Miden-->>BridgeAPI: Public P2ID note committed and consumable
    BridgeAPI->>BridgeAPI: Persist Miden mint tx id and mark quote SUCCESS
    Note over BridgeAPI,Recipient: Bridge success means the payout note exists
    Recipient->>Miden: Sync public note targeted to recipient account
    Recipient->>Miden: Consume and claim P2ID note
    Miden-->>Recipient: Recipient balance updated

Outbound: Miden public note to EVM release

sequenceDiagram
    participant User
    participant BridgeAPI as Bridge API
    participant Miden
    participant Poller
    participant Anvil

    User->>BridgeAPI: Request quote miden-testnet:eth to eth-anvil:eth
    BridgeAPI-->>User: Return bridge account plus BridgeOutV1 memo
    User->>Miden: Create public BridgeOutV1 note with assets
    Poller->>Miden: Sync and scan public notes
    Poller->>Poller: Validate quote hash, target account, faucet, amount
    Poller->>Miden: Consume matched note
    Miden-->>Poller: Consume tx committed
    Poller->>Anvil: Release funds to EVM recipient
    Poller->>BridgeAPI: Persist tx ids and mark SUCCESS

Restart recovery

flowchart TD
    A["Quote is PROCESSING"] --> B["Bridge container restarts"]
    B --> C["Reload non-terminal quote from Postgres"]
    C --> D{"Durable Miden tx id exists?"}
    D -->|Yes| E["Wait for existing tx confirmation"]
